以下のコードをレビューしてください: $ARGUMENTS

## レビュー観点

- **バグ・ロジックの問題**: 潜在的なバグや誤ったロジックがないか
- **セキュリティ**: XSS・SQLインジェクション・認証漏れなどのリスクがないか
- **パフォーマンス**: 不要な再レンダリング・重い処理・N+1問題などがないか
- **可読性**: 変数名・関数名が適切か、複雑すぎる処理がないか
- **型安全性**: TypeScriptの型が適切に使われているか

## 出力形式

問題点は重要度別に分類して報告する:
- 🔴 **Critical**: 必ず修正すべき問題
- 🟡 **Warning**: できれば修正したい問題
- 🟢 **Suggestion**: 改善提案（必須ではない）

問題がなければ「問題なし」と報告する。
